2026-06-26

🔴 Critical: 3 · 🟡 Important: 3 · 🟢 Opportunities: 4

Primary: Emil Kowalski — Restraint & Speed

The dashboard's motion system is more disciplined than most — a short duration range (80–280ms), purpose-built custom easings, and a deliberate design law ("clay never carries data") give it a strong skeleton. The biggest problems are two structural gaps: the verdict hero's entrance uses attest-fade-up, which applies filter: blur(3px) to a flat-data surface in direct violation of the project's own aesthetic law; and settings tab content swaps with a hard zero-duration snap, which is jarring for a panel toggled multiple times per session. Stat cards each get an independent entrance via attest-enter but no --stagger-index offsets, so all four cards animate simultaneously, reading as a single undifferentiated burst rather than a staged arrival. The RunMetadata card at the bottom of RunDetail is the only card without any entrance, creating an inconsistency. Skeleton components use ease-in-out — correct for opacity pulses — but the transition from skeleton to real content is an instant swap with no crossfade. These are fixable in a single focused pass.

Issues to Address

🔴 Critical

Stat cards all animate simultaneously — stagger-spam without offsets

Each StatCard applies className="attest-enter" but passes no --stagger-index. All cards start their 200ms fade-up at the same time, producing a simultaneous burst. Per Emil, this is worse than no animation: it creates a perceptible UI hiccup on every home page visit without any orientation benefit. The frequency gate says daily-use elements should get subtle, fast motion or none at all. Four identical simultaneous fades reads as a loading artifact, not a purposeful reveal. Fix: remove attest-enter from StatCards entirely, or add stagger offsets of 0/1/2/3 on --stagger-index and reduce to a very subtle translateY of 4px with 140ms duration.

🔴 Critical

Skeleton-to-content transition is an instant hard swap

Both HomePlaceholder and RunDetail use runsPending ? <Skeleton /> : <content /> conditionally. When data arrives, the skeleton disappears and content appears in a single render frame. There is no crossfade, no short opacity transition. This reads as a flash — users see a white/dark pop. The fix per Emil: wrap the conditional in a keyed container and apply transition-state (the existing opacity var(--dur-3) ease-out utility in globals.css) to both elements, or at minimum apply animation: attest-fade-up var(--dur-3) var(--ease-out) both on the content element on mount.

Issues to Address

🟡 Important

VerdictHero uses blur entrance on a flat-data surface

The completed verdict panel applies className="attest-enter", which runs attest-fade-up: filter: blur(3px) animating to 0. The project's own design law (ui-context.md §9.2) explicitly prohibits "decorative motion on flat-data surfaces." Blur is a Jakub signal for "materializing" — it communicates "this element is coming into focus." On a verdict panel containing a machine-readable judgment (PASSED/FAILED), the blur implies the result is still uncertain, which directly undermines the "confident settle" the user expects. Fix: replace with an opacity-only or opacity + minimal translateY (4px, no blur) entrance using --ease-out-expo for a faster, more authoritative settle. The entrance should feel like a readout appearing, not a card materializing.

🟡 Important

Skeleton pulse uses ease-in-out instead of a sine-shaped curve

Skeleton.tsx fires attest-skeleton-pulse at animation-timing-function: ease-in-out (the default, inherited from globals). The pulse keyframe (0%/100% opacity:1 → 50% opacity:0.45) with ease-in-out produces an asymmetric feel — it decelerates into the dim state and accelerates out, rather than a smooth sinusoidal breath. For a background loading indicator, a symmetric sine curve reads as more natural and less mechanical. Fix: declare animation-timing-function: cubic-bezier(0.37, 0, 0.63, 1) on the skeleton element, which approximates a sine and produces a perfectly even breath. Duration 1.8s is appropriate.
